The validated as-of-connection-date method (from the quick-task background):
  1. Navigate to the connection's /in/<slug>/details/experience/ page (full experience history), NOT just the top experience card.
  2. Read the contact's linkedinConnectionDate via GET /api/contacts/<id> (field linkedinConnectionDate).
  3. Parse each experience entry's date range (start–end; "Present" = ongoing). LinkedIn groups multiple roles under one company header — handle grouped sub-roles.
  4. Select the role whose date span CONTAINS the connection date. For a grouped company entry, use the matching sub-role title + the parent company name.
  5. Fallbacks:
    • Connection date PREDATES earliest listed role → write null for both fields, log the limitation (e.g. "profile history starts , predates connection date").
    • Connection date lands in an employment GAP → use closest prior role and flag it.
    • MULTIPLE roles overlap (concurrent) → pick the primary/full-time role, note concurrency.
  6. Write back via PATCH /api/contacts/<id>/enrichment with companyAtConnection + roleAtConnection (null allowed). Endpoint stamps enrichmentStatus='enriched'.
