# Phase quick-260611-l44 Plan 01: Clickable Contact Names + Override Button in Recommendations Summary

**One-liner:** Contact names in recommendation cards now link to `/dashboard/contacts/${contactId}` and each card has an Override button that PUTs `doNotUseForIntros=true` and optimistically removes the card on success.

## What Was Built

**RecommendationCard (`recommendation-card.tsx`):**
- Added `contactId: string`, `onOverride?: () => void`, `overriding?: boolean` to `RecommendationCardProps`
- Replaced plain `<span className='font-medium'>{contactName}</span>` with `<Link href={/dashboard/contacts/${contactId}} className='font-medium hover:underline'>` using `next/link`
- Added Override `<Button variant='outline' size='sm'>` rendered after the LinkedIn badge when `onOverride` is provided; wired `onClick`, `disabled={overriding}`, `tabIndex={-1}`
- Imported `Button` from `@/components/ui/button` and `Link` from `next/link`

**RecommendationList (`recommendation-list.tsx`):**
- Added `overridingIds: Set<string>` state to track in-flight override requests per contact
- Added `handleOverride` useCallback: marks contactId as in-flight, calls `PUT /api/contacts/${contactId}` with `{ doNotUseForIntros: true }`, optimistically removes card on success (`setRecommendations` filter), shows `toast.success`; on failure shows `toast.error` and leaves card in place; clears in-flight id in finally
- Imported `toast` from `sonner`
- Passed `contactId={rec.contact.id}`, `onOverride={() => handleOverride(rec.contact.id)}`, `overriding={overridingIds.has(rec.contact.id)}` to each RecommendationCard
