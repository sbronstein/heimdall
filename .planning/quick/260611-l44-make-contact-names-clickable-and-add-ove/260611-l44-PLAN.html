In the job-leads intro-matches UI (RecommendationCard on the job lead drilldown page), make each contact name a clickable link to that contact's record page, and add an "Override" button next to the LinkedIn badge that excludes the contact from future intro recommendations by setting `doNotUseForIntros=true`.

Purpose: Lets the owner jump from a recommendation straight to the contact record, and prune contacts they don't want surfaced as intro paths — buildRecommendations already hard-excludes flagged contacts, so the override takes effect on the next load and is reflected immediately via optimistic removal.

Output: Updated RecommendationCard (name link + Override button) and RecommendationList (contactId threading + override handler).

Task 2: Thread contactId and wire override handler in RecommendationList src/features/job-leads/components/recommendation-list.tsx Import `{ toast } from 'sonner'`.

Add contactId={rec.contact.id} to the RecommendationCard props in the map.

Add a handleOverride callback (useCallback) that takes contactId: string and:

  1. Tracks an in-flight set of contact ids in state (e.g. overridingIds) so the card's Override button can be disabled; pass overriding={overridingIds.has(rec.contact.id)} to the card.
  2. Calls await fetch('/api/contacts/${contactId}', { method: 'PUT', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ doNotUseForIntros: true }) }).
  3. On success (res.ok and json.success): optimistically remove the contact from local state with setRecommendations((prev) => prev?.filter((r) => r.contact.id !== contactId) ?? null) and toast.success('Contact excluded from intro recommendations').
  4. On failure: toast.error(json.error || 'Failed to override contact') and leave the card in place; clear the in-flight id in a finally block.

Pass onOverride={() => handleOverride(rec.contact.id)} to each RecommendationCard.

Follow existing conventions: REST-only mutation (no server action), single quotes, 2-space indent, named export preserved. npx tsc --noEmit 2>&1 | grep -i recommendation-list || echo "no type errors in recommendation-list" RecommendationList passes contactId to each card, and Override triggers PUT /api/contacts/[id] with { doNotUseForIntros: true }; on success the card is removed from the list with a success toast, on failure a toast is shown and the card stays.
