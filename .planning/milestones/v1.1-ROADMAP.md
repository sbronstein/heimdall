# Milestone v1.1: LinkedIn Scraping by Company

**Status:** ✅ SHIPPED 2026-05-20
**Phases:** 7-10
**Total Plans:** 11

## Overview

Extends the `scrape-linkedin-connections` skill so it can scrape 2nd-degree connections at any target company — not just at companies attached to a specific LinkedIn job posting — by reusing the existing `job_leads` + `prospects` tables via "synthetic" leads where `linkedinJobUrl` is null. The milestone also folded in a late-added enrichment phase (Phase 10) that surfaces each connection's company and role *at the time of connection* in the triage flow, backfilled by a paced agent-browser sweep and just-in-time enrichment during company-scope triage.

Theme arc: schema + API foundation (Phase 7) → skill input parsing, navigation branching, drain (Phase 8) → UI for company-scope leads (Phase 9) → at-connection enrichment for triage (Phase 10, inserted 2026-05-20).

## Milestone Summary

**Decimal / Inserted Phases:**

- Phase 10: Connection Company + Role Enrichment for Triage — added 2026-05-20 mid-milestone to surface at-connection company/role in triage and backfill the CSV gap with a paced agent-browser sweep.

**Key Decisions:**

- Synthetic `job_leads` row (`linkedinJobUrl = null`, `roleTitle = null` or "Company-wide scrape") rather than a new entity or table — reuse existing routes / UI / state machine.
- No role filter for company scope — the skill returns all 2nd-degree connections at the company.
- Bare-company-name disambiguation handled inline by the skill (top 3–5 matches: name + employee count + industry) rather than in the app.
- Single shared drain queue (`GET /api/job-leads?status=queued`); skill navigation branches on whether `linkedinJobUrl` is null — no separate queue, status value, or route.
- Phase ordering Schema+API (7) → Skill (8) → UI (9), because both skill and UI depend on the nullable-column shape and synthetic-lead creation route.
- Just-in-time enrichment on the company-triage path does **no** inline DB write (Variant per Plan 10-03) — keeps the read path pure.

**Issues Resolved:**

- Company-scope leads now flow through the entire existing pipeline (create → drain → scrape → prospects → recommendations) without special-casing.
- At-connection company/role is captured at CSV-import time and backfillable for the legacy 1000+ contact backlog via the paced sweep.

**Issues Deferred (carried forward):**

- Open UAT scenarios: Phase 6 (1), Phase 8 (4) — partial human UAT.
- Verification sign-off `human_needed`: Phases 1, 6, 8; Phase 9 verification `gaps_found`.
