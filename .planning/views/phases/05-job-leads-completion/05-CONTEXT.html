Heimdall · gathered 2026-05-13 · Ready for planning · Mode: discuss

Phase Boundary

The LinkedIn job-leads scraper runs cleanly end-to-end. Hardcoded debug strings replaced with the lead's actual company name. Heavy LinkedIn pages use waitUntil: 'domcontentloaded' + targeted waitForSelector, not fixed waitForTimeout. A scrape that hangs or fails is bounded by a 90-second wall-clock timeout and reverts searching → scraped. Errors are surfaced to the UI via new lastError / lastErrorAt columns AND a job_lead_search_failed timeline event. Detail page renders a banner with Retry. Debug noise trimmed to ~5 navigation breadcrumbs. Browser context closed after successful scrapes in remote (CDP/WS) mode only.

Implementation Decisions

Error Surfacing Model (JL-A5)

  • D-01 — Both column + timeline event. New columns last_error: text and last_error_at: timestamp. Also write job_lead_search_failed timeline event per failure.
  • D-02 — Inline banner on detail page + Retry button. No toast. Banner sits above Find Connections.
  • D-03 — Format: <category>: <first 200 chars of err.message>. Five categories: Timeout, LinkedIn navigation failed, No prospects found, Browser unavailable, Unknown error.
  • D-04 — lastError/lastErrorAt cleared on every fresh /search POST. No separate "dismiss" UI.
  • D-05 — /api/job-leads/[id]/status returns { status, prospectCount, lastError, lastErrorAt, updatedAt }. Single round-trip per poll tick.

Timeout Budget + Revert Behavior (JL-A4)

  • D-06 — 90 seconds wall-clock budget. No env override.
  • D-07 — Timeout label: Timeout: scrape exceeded 90s.
  • D-08 — Status revert: searching → scraped (per SC #2). Not pending, not failed.
  • D-09 — Same revert flow applies to non-timeout failures (any thrown error in the catch block).
  • D-10 — Promise.race + leave Playwright dangling. No AbortController, no context.close() on timeout. Tension flag for plan-checker: SC #3 ("no leaked browser after completion") interpreted as "successful completion closes; timeout-abort releases ownership and lets context GC."
  • D-11 — Timeout wrapper inline in search/route.ts. Custom ScrapeTimeoutError class so the catch block can distinguish Timeout from other failures.

Browser Lifecycle (JL-A3 / SC #3)

  • D-12 — Close context in remote (CDP/WS) mode only. Local launchPersistentContext stays alive between scrapes (relaunch cost).
  • D-13 — Failed-scrape close matches success path: remote → close; local → don't.
  • D-14 — Signature change: scrapeConnections returns just { prospects }; context lifecycle is internal. Acceptable — one caller.

Debug-Log Cleanup Scope (JL-A3 / SC #3)

  • D-15 — Keep ~5 navigation breadcrumbs, drop DOM dumps. Keep: "Navigating to job posting...", "On company page: <url>", "Navigated to people search" / "Clicked employees link", "Scraping page N", "Scraping complete. Found X prospects at <company>". Drop: JSON.stringify(pageLinks), JSON.stringify(allLinks), per-page chatter, strategy-debug logs. Failure logs upgraded from console.log to thrown errors so the categorizer can pick them up.
  • D-16 — console.error stays unconditional in catch paths.
  • D-17 — No DEBUG_SCRAPE env flag. Permanent slim breadcrumbs.

  • D-19 — Filesystem grep assertion that 'point' is gone from scrape-connections.ts. Mirrors Phase 4 D-16 pattern.

Specific Ideas

  • ScrapeTimeoutError class — one-liner. Catch checks err.name === 'ScrapeTimeoutError'.
  • Categorizer — ~15 lines, regex-based: 
    function categorizeError(err: unknown): string {
  if (err instanceof ScrapeTimeoutError) return `Timeout: ${err.message}`;
  const msg = err instanceof Error ? err.message : String(err);
  const tail = msg.slice(0, 200);
  if (/navigat|company link|employees link/i.test(msg))
    return `LinkedIn navigation failed: ${tail}`;
  if (/no prospects|0 prospects|no results/i.test(msg))
    return `No prospects found: ${tail}`;
  if (/CDP|WS|connect|launch|Chromium|browser/i.test(msg))
    return `Browser unavailable: ${tail}`;
  return `Unknown error: ${tail}`;
}
  • Integration test pattern — vi.useFakeTimers({ shouldAdvanceTime: true }); vi.advanceTimersByTimeAsync(90_000); await new Promise(r => setImmediate(r)); then assert DB state.
  • Both new columns are NULLABLE with no default. NULL = no error to surface.
  • No new indexes — /status reads by primary key. Phase 6 (PERF-A4) owns global index strategy.
