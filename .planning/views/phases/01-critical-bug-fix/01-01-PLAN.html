Objective

Eliminate the React hydration crash (BUG-01) that breaks sidebar navigation after large LinkedIn imports, and guard the unsafe emailAddresses[0] array access (BUG-02) that can crash on edge-case Clerk users.

Per CONTEXT.md decisions D-06 and D-07, the working tree already contains partial fixes for BUG-01 — user-avatar-profile.tsx root is <span>, inner showInfo is <span>, and app-sidebar.tsx already removed the {user && ...} guards at lines 148 and 166, passing user={user ?? null} through. This plan audits those edits against bug.md's prescription, applies the two remaining BUG-02 optional-chain guards, performs a dev-server smoke verification (D-03), and produces surgical commits scoped to the three BUG files only (D-09, D-10).
Recommended commit shape (D-12): Two atomic commits — one for BUG-01 (span migration + unconditional render), one for BUG-02 (optional-chain guards) — using git add -p on user-avatar-profile.tsx to split the BUG-01 and BUG-02 hunks. Fallback: single combined commit (executor discretion).
