Phase 3 · Plan 02: Strip starter-template auth artifacts

Heimdall · planned 2026-05-12 · Pending execution · Wave 1 · Requirement: SEC-A2

Objective

Strip the two starter-template auth artifacts that leak third-party network calls and present a misleading auth control to users: (a) delete the no-op github-auth-button.tsx component and remove every render/import of it, (b) remove the await fetch('https://api.github.com/repos/...') call from both auth page.tsx files, (c) remove the now-dead stars prop and the <Link>...Star on GitHub...{stars}</Link> block from both view components.
Purpose. Close SEC-A2 — the "Continue with GitHub" button is a no-op (its onClick just logs to console — verified in github-auth-button.tsx:16) and presenting it as a working auth method on a single-user Clerk-locked app is a UX/security smell. The api.github.com fetch is starter-template residue that issues an outbound request to a third-party API on every render of the sign-in/sign-up pages — pointless in this fork, an SSRF-shaped surface to delete.
