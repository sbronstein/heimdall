Objective

Activate the dead clerkMiddleware() file by renaming src/proxy.ts to src/middleware.ts (which Next.js auto-loads), expand the protected-route matcher to cover /api/(.*) in addition to /dashboard(.*), and add an explicit 401-envelope short-circuit for unauthenticated /api/* requests so the API-V1 { success, data, error, meta } contract is preserved for CLI consumers.
Purpose. Close SEC-A1 — every /api/* route currently authenticates against zero callers. After this plan, the middleware is the single chokepoint enforcing both Clerk session presence and the steve@bronstein.org email lock on every API path. Per D-06, no per-route auth() calls are added inside src/app/api/**/route.ts — middleware-only enforcement to avoid drift across 32 route files.
