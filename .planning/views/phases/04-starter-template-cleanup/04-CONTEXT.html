⚠ Critical Pre-Discovery

  • PD-01: Every name in DEBT-A1..A5 still exists in the tree — no requirement is already satisfied.
  • PD-02: infobar.tsx is NOT isolated — deleting just that file breaks the build. 5+ files depend on InfobarProvider/InfobarContent. Transitive removal is mandatory.
  • PD-03: Kanban removal has TWO surfaces: the route AND the feature folder (src/features/kanban/ with its zustand+localStorage anti-pattern store). Both must go.
  • PD-04: src/features/products/ blast radius is wider than DEBT-A1 names: mock-api.ts, three exports in data.ts, breadcrumb entry are all products-only dead code.
  • PD-05: SC #1 (404 verification) does not require an HTTP smoke. In Next.js App Router, deleting a route directory IS the 404. Filesystem-existence assertion + clean build is a stronger contract.
