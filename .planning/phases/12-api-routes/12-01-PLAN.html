Deliver the campaign-lifecycle REST surface: list (with per-campaign progress counts), create, read-single, update, and soft-delete for `/api/outreach-campaigns`. This is the entry point the Phase 14 builder UI and the CLI use to create and track campaigns.

Purpose: CAMP-06 (create named campaign with goal → get id) and CAMP-08 (list campaigns with selected/generated/approved/drafted counts) are the user-facing "I can start and monitor a campaign" capability. Output: Two new route files, both returning the { success, data, error, meta } envelope, validated with Zod, every write logged to timeline_events.
