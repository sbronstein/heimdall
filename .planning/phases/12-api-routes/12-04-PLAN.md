<objective>
Deliver the two endpoints the Phase 16 `generate-outreach-emails` skill uses: a bulk read that hands the skill everything it needs to write all pending emails in one round-trip (campaign goal + per-contact brief + recent interactions + a low-context flag), and the write-back that stores the generated subject/body. This replaces the N+1 per-email design the ARCHITECTURE.md draft proposed (D-01), honoring the project's anti-N+1 constraint at 1500+ contacts.

Purpose: GEN-05 — all generation runs in the skill; the API only enqueues pending rows (Plan 02) and accepts write-back. The generation-context endpoint is the "enqueued context" read; the generation route is the write-back.
Output: One aggregated-read route + one write-back route, envelope-compliant, with the write logged to timeline.
</objective>
