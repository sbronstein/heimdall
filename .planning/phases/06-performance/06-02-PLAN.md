From drizzle-orm (neon-http adapter, v0.45.1):
```typescript
// Per CONTEXT D-02: the neon-http adapter supports db.transaction(); statements
// inside the callback are coalesced into a single transactional HTTP request.
// Every Drizzle call inside the callback MUST use the `tx` parameter, NOT the
// imported `db`, or atomicity does not hold.
db.transaction<T>(callback: (tx: NeonHttpTransaction<...>) => Promise<T>): Promise<T>;
```

The `tx` handle is NOT `typeof db` — it is `NeonHttpTransaction<...>`, which forbids nested transactions (`tx.transaction(...)` is a compile error). This narrower type is what matchConnections receives.
