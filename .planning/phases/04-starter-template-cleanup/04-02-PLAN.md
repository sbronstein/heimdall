<objective>
Execute DEBT-A2 — delete the four starter dashboard routes (`exclusive`, `workspaces`, `workspaces/team`, `billing`) AND the entire Infobar UI machinery that was wired into the dashboard layout. The 805-line `infobar.tsx` cannot be deleted in isolation: it's wired into `dashboard/layout.tsx` via `InfobarProvider` and `<InfoSidebar />`, and its `InfobarContent` type flows through the `infoContent` prop on `PageContainer` and `Heading`. Per D-04, this is a **transitive** removal — surgical deletion of `infobar.tsx` alone breaks the build.

Purpose: The four starter routes are Clerk-org / billing / workspace demo pages with no relationship to the Heimdall job-search CRM (verified by CONCERNS.md §"Starter Template Infobar / Workspaces / Billing / Exclusive Pages"). The Infobar machinery exists solely to support these four routes' contextual right-side help panel — scout confirmed every `infoContent={...}` call site is one of these four pages. After this plan lands, the dashboard layout simplifies to `<KBar><SidebarProvider><AppSidebar /><SidebarInset>...</SidebarInset></SidebarProvider><SearchCommand /></KBar>` (per D-06: no replacement panel).

Output: 4 deleted route directories, 4 deleted machinery files (`infobar.tsx`, `info-button.tsx`, `info-sidebar.tsx`, `infoconfig.ts`), 3 edited files (`dashboard/layout.tsx`, `page-container.tsx`, `heading.tsx`).

**Edit ordering is load-bearing** — see Task 1 — the build will go red mid-plan if the order is wrong.
</objective>
