## Component Variants

### Discriminator (both components)

```tsx
const isCompanyScope = lead.linkedinJobUrl === null;
```

Declare this as the first line of the component body, before any JSX. Key ALL conditional branches off this single boolean. Never key off `lead.roleTitle === 'Company-wide scrape'` or any sentinel string check.

### JL-C8: Detail Page тАФ `scrape-results.tsx`

**Current layout (job-URL lead тАФ unchanged):**
- `CardTitle` (16px / `text-base` / weight semibold): `lead.roleTitle || 'Unknown Role'`
- `<p className="text-muted-foreground text-sm">`: `lead.companyName || 'Unknown Company'`
- "View Job Posting" link: already null-guarded, no change

**New layout for company-scope lead (`isCompanyScope === true`):**

Replace the `CardTitle` slot and subtitle line with the following structure:

```tsx
<CardHeader className='pb-3'>
  <div className='flex items-center gap-2'>
    <CardTitle className='text-base'>
      {lead.companyName || 'Company scrape'}
    </CardTitle>
    <Badge variant='secondary'>Company scrape</Badge>
  </div>
  {/* role-title line is intentionally absent for company-scope leads */}
</CardHeader>
```

Rules:
- `CardTitle` size: `text-base` (16px, weight inherits from shadcn `CardTitle` = semibold 600). Do not add inline font-weight overrides.
- Badge copy: exactly **"Company scrape"** (two words, no hyphen, sentence case).
- Badge variant: `secondary` тАФ maps to `bg-secondary text-secondary-foreground`, rendering a calm muted chip that does not compete with the colored status badge on the list row.
- Badge placement: inline after the company name text, within a `flex items-center gap-2` wrapper that wraps gracefully at narrow widths (wrapping is acceptable тАФ the badge is not critical-path information).
- Fallback when `lead.companyName` is null: render `'Company scrape'` as the CardTitle text. The badge still appears next to it. This produces the heading "Company scrape [Company scrape badge]" тАФ acceptable because the badge is secondary context and the heading text carries the primary meaning. Never render "Unknown Role" for a company-scope lead.
- Employee-count line: OMIT entirely. Out of scope per CONTEXT.md `<deferred>`. Do not render a blank line or "undefined employees".
- "View Job Posting" link: already null-guarded at line 32 (`{lead.linkedinJobUrl && (...)}`). Verify this guard exists; do not re-add it.

**Unchanged for job-URL leads (`isCompanyScope === false`):**
- `CardTitle`: `lead.roleTitle || 'Unknown Role'`
- subtitle `<p>`: `lead.companyName || 'Unknown Company'`
- "View Job Posting" link: conditionally shown as before

### JL-C9: List/Card Row тАФ `job-lead-card.tsx`

**Current layout (every lead тАФ job-URL leads unchanged):**
- Leading icon: `<IconBuilding className='text-muted-foreground h-4 w-4 shrink-0' />`
- Company name: `lead.companyName || 'Unknown Company'`
- Role subtitle: `{lead.roleTitle && (<p className='text-muted-foreground mt-1 truncate text-sm'>{lead.roleTitle}</p>)}`
- Right side: prospect count, status badge (`Badge variant='outline'`), date

**New layout for company-scope lead (`isCompanyScope === true`):**

Change 1 тАФ Icon swap:
```tsx
{isCompanyScope
  ? <IconBuildingCommunity className='text-muted-foreground h-4 w-4 shrink-0' />
  : <IconBuilding className='text-muted-foreground h-4 w-4 shrink-0' />
}
```
- Use `IconBuildingCommunity` (confirmed exported from `@tabler/icons-react` v3.38.0). This icon shows two buildings overlapping, contrasting with `IconBuilding`'s single-building silhouette.
- All icon classes identical to existing: `text-muted-foreground h-4 w-4 shrink-0`. No size deviation.

Change 2 тАФ Role subtitle suppression for company-scope:
```tsx
{!isCompanyScope && lead.roleTitle && (
  <p className='text-muted-foreground mt-1 truncate text-sm'>{lead.roleTitle}</p>
)}
```
- The sentinel string `'Company-wide scrape'` must never render as a visible subtitle. Suppressing the entire subtitle line for `isCompanyScope` leads achieves this cleanly.

Change 3 тАФ "Company" pill in the right-side badge group:
```tsx
<div className='flex items-center gap-3'>
  {isCompanyScope && (
    <Badge variant='outline'>Company</Badge>
  )}
  {lead.prospectCount > 0 && (
    <div className='text-muted-foreground flex items-center gap-1 text-sm'>
      <IconUsers className='h-3.5 w-3.5' />
      {lead.prospectCount}
    </div>
  )}
  <Badge
    variant='outline'
    className={statusColors[lead.status] || ''}
  >
    {lead.status.replace(/_/g, ' ')}
  </Badge>
  <span className='text-muted-foreground text-xs'>
    {new Date(lead.createdAt).toLocaleDateString()}
  </span>
</div>
```

Rules:
- "Company" pill copy: exactly **"Company"** (one word, sentence case). Shorter than "Company scrape" to fit the dense row.
- Pill variant: `outline` тАФ same variant as the status badge, using `text-foreground` and a border. This keeps the pill visually calm at the same weight level as the status badge without adding another filled color.
- Pill placement: leftmost item in the right-side `flex items-center gap-3` cluster, before the prospect count and before the status badge. This matches the scan order (type signal тЖТ prospect signal тЖТ status signal тЖТ date).
- The `gap-3` on the right-side container is already in the existing code at line 40 тАФ no gap change needed.

**Unchanged for job-URL leads (`isCompanyScope === false`):**
- `IconBuilding` as the leading icon
- Role subtitle conditionally rendered as before
- No "Company" pill

Phase 9 introduces no new type styles. All text in the new branches inherits existing classes from the surrounding card chrome.

| Role | Tailwind | px equiv | Weight | Line Height | Where used |
|------|----------|----------|--------|-------------|------------|

| Date | `text-xs` | 12px | 400 (regular) | n/a | Card row date (`job-lead-card.tsx` line 54) |

Maximum 4 sizes, 2 weights (regular 400, semibold 600). The badge `font-medium` (500) is a shadcn primitive тАФ do not override it.
