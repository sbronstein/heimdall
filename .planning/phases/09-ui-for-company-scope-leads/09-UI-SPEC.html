Phase 9 — UI Design Contract

Visual and interaction contract for Phase 9: UI for Company-Scope Leads. Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor. All decisions sourced from 09-CONTEXT.md (D-01 through D-03, CD-01 through CD-04) and direct inspection of the two target components.

Design System

Property Value

Icon library @tabler/icons-react v3.38.0

Component Variants

Discriminator (both components)

const isCompanyScope = lead.linkedinJobUrl === null;

Declare this as the first line of the component body, before any JSX. Key ALL conditional branches off this single boolean. Never key off lead.roleTitle === 'Company-wide scrape' or any sentinel string check.

JL-C8: Detail Page — scrape-results.tsx

Current layout (job-URL lead — unchanged):

  • CardTitle (16px / text-base / weight semibold): lead.roleTitle || 'Unknown Role'
  • <p className="text-muted-foreground text-sm">: lead.companyName || 'Unknown Company'
  • "View Job Posting" link: already null-guarded, no change

New layout for company-scope lead (isCompanyScope === true):

Replace the CardTitle slot and subtitle line with the following structure:

<CardHeader className='pb-3'>
  <div className='flex items-center gap-2'>
    <CardTitle className='text-base'>
      {lead.companyName || 'Company scrape'}
    </CardTitle>
    <Badge variant='secondary'>Company scrape</Badge>
  </div>
  {/* role-title line is intentionally absent for company-scope leads */}
</CardHeader>

Rules:

  • CardTitle size: text-base (16px, weight inherits from shadcn CardTitle = semibold 600). Do not add inline font-weight overrides.
  • Badge copy: exactly "Company scrape" (two words, no hyphen, sentence case).
  • Badge variant: secondary — maps to bg-secondary text-secondary-foreground, rendering a calm muted chip that does not compete with the colored status badge on the list row.
  • Badge placement: inline after the company name text, within a flex items-center gap-2 wrapper that wraps gracefully at narrow widths (wrapping is acceptable — the badge is not critical-path information).
  • Fallback when lead.companyName is null: render 'Company scrape' as the CardTitle text. The badge still appears next to it. This produces the heading "Company scrape [Company scrape badge]" — acceptable because the badge is secondary context and the heading text carries the primary meaning. Never render "Unknown Role" for a company-scope lead.
  • Employee-count line: OMIT entirely. Out of scope per CONTEXT.md <deferred>. Do not render a blank line or "undefined employees".
  • "View Job Posting" link: already null-guarded at line 32 ({lead.linkedinJobUrl && (...)}). Verify this guard exists; do not re-add it.

Unchanged for job-URL leads (isCompanyScope === false):

  • CardTitle: lead.roleTitle || 'Unknown Role'
  • subtitle <p>: lead.companyName || 'Unknown Company'
  • "View Job Posting" link: conditionally shown as before

JL-C9: List/Card Row — job-lead-card.tsx

Current layout (every lead — job-URL leads unchanged):

  • Leading icon: <IconBuilding className='text-muted-foreground h-4 w-4 shrink-0' />
  • Company name: lead.companyName || 'Unknown Company'
  • Role subtitle: {lead.roleTitle && (<p className='text-muted-foreground mt-1 truncate text-sm'>{lead.roleTitle}</p>)}
  • Right side: prospect count, status badge (Badge variant='outline'), date

New layout for company-scope lead (isCompanyScope === true):

Change 1 — Icon swap:

{isCompanyScope
  ? <IconBuildingCommunity className='text-muted-foreground h-4 w-4 shrink-0' />
  : <IconBuilding className='text-muted-foreground h-4 w-4 shrink-0' />
}
  • Use IconBuildingCommunity (confirmed exported from @tabler/icons-react v3.38.0). This icon shows two buildings overlapping, contrasting with IconBuilding's single-building silhouette.
  • All icon classes identical to existing: text-muted-foreground h-4 w-4 shrink-0. No size deviation.

Change 2 — Role subtitle suppression for company-scope:

{!isCompanyScope && lead.roleTitle && (
  <p className='text-muted-foreground mt-1 truncate text-sm'>{lead.roleTitle}</p>
)}
  • The sentinel string 'Company-wide scrape' must never render as a visible subtitle. Suppressing the entire subtitle line for isCompanyScope leads achieves this cleanly.

Change 3 — "Company" pill in the right-side badge group:

<div className='flex items-center gap-3'>
  {isCompanyScope && (
    <Badge variant='outline'>Company</Badge>
  )}
  {lead.prospectCount > 0 && (
    <div className='text-muted-foreground flex items-center gap-1 text-sm'>
      <IconUsers className='h-3.5 w-3.5' />
      {lead.prospectCount}
    </div>
  )}
  <Badge
    variant='outline'
    className={statusColors[lead.status] || ''}
  >
    {lead.status.replace(/_/g, ' ')}
  </Badge>
  <span className='text-muted-foreground text-xs'>
    {new Date(lead.createdAt).toLocaleDateString()}
  </span>
</div>

Rules:

  • "Company" pill copy: exactly "Company" (one word, sentence case). Shorter than "Company scrape" to fit the dense row.
  • Pill variant: outline — same variant as the status badge, using text-foreground and a border. This keeps the pill visually calm at the same weight level as the status badge without adding another filled color.
  • Pill placement: leftmost item in the right-side flex items-center gap-3 cluster, before the prospect count and before the status badge. This matches the scan order (type signal → prospect signal → status signal → date).
  • The gap-3 on the right-side container is already in the existing code at line 40 — no gap change needed.

Unchanged for job-URL leads (isCompanyScope === false):

  • IconBuilding as the leading icon
  • Role subtitle conditionally rendered as before
  • No "Company" pill

Typography

Phase 9 introduces no new type styles. All text in the new branches inherits existing classes from the surrounding card chrome.

Color

Phase 9 introduces no new colors. All new elements use existing shadcn CSS variable tokens. Color budget: ~60% background / ~30% card chrome / ~10% accent (badge fills + borders).

Role CSS variable Semantic usage in this phase
Dominant (60%) --background Card background, page surface
Secondary (30%) --card Card chrome (<Card> component)
Accent — badge fills --secondary / --secondary-foreground "Company scrape" badge fill on detail page (variant secondary)
Accent — badge border --border / --foreground "Company" pill border on list row (variant outline)
Muted text --muted-foreground Leading icon, prospect count, date — unchanged

Accent reserved for:

  • "Company scrape" badge on the detail page header (variant='secondary')
  • "Company" pill on the list row (variant='outline')
  • Status badges on list rows (already existing — not introduced by this phase)

No new color tokens. No new bg-* or text-* Tailwind color classes. Dark-mode variants are handled automatically by the existing shadcn CSS variable system.
