## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer→source repo | Hand-edited schema + migration files; reviewed via PR/commit before merge |
| Drizzle Kit→`drizzle/migrations/*.sql` | Auto-generated DDL; reviewed by human before `db:migrate` |
| `npm run db:migrate`→live Postgres | DDL applied against `DATABASE_URL` (Neon dev branch); single-statement, non-destructive |
| Vitest harness→PGlite | In-process Postgres for the regression test; no network egress |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | Drizzle Kit emitted SQL (could over-generate, e.g., recreate the entire `job_leads` table and drop data) | mitigate | Task 3 explicitly requires inspecting the emitted SQL; the `[BLOCKING]` tag pauses execution if the file contains `DROP TABLE`, `CREATE TABLE "job_leads"`, or references to columns other than `linkedin_job_url` — verified by the grep in Task 3's acceptance criteria |
| T-07-02 | Denial of Service | `npm run db:migrate` failure on live Postgres (lock contention, connection drop) | accept | Single-user dev DB; non-destructive ALTER COLUMN DROP NOT NULL takes a brief AccessExclusiveLock; rollback is trivial (re-add NOT NULL — though no data would yet violate it). Risk is low and self-bounded. |
| T-07-03 | Tampering | Schema regression test could be deleted later without anyone noticing | accept | Phase 2's test infrastructure runs the full Vitest suite in pre-push hook; a deleted phase-N test reduces total count but Vitest does not fail on "fewer tests than before". This is a known limitation; not blocking Phase 7. |
| T-07-04 | Information Disclosure | Migration runbook header comment exposes intent in source control | accept | Comment is informational; no secrets or internal-only data. Public-by-default OSS-style repo hygiene. |
| T-07-05 | Repudiation | Migration applied to live DB without traceable record | mitigate | `drizzle/migrations/meta/_journal.json` records the migration tag + hash; git commit on the migration file provides the source-control audit trail per `commit_docs=true` |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan; `drizzle-kit`, `vitest`, `@neondatabase/serverless` are pre-existing dependencies. Package legitimacy gate N/A. |
</threat_model>
