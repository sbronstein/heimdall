<objective>
Pin the D-17 invariant: the existing `PATCH /api/job-leads/[id]/status` and `POST /api/job-leads/[id]/prospects` routes are input-shape agnostic — they operate on `id` lookups, never read or branch on `linkedinJobUrl`, and therefore must work identically against company-scope leads (`linkedinJobUrl: null`). No production code changes; pure regression tests that lock the behavior. Without these tests, a future refactor could add an accidental `if (lead.linkedinJobUrl)` guard that silently breaks the company-scope drain path — the skill's main loop assumes both routes work uniformly across lead shapes.

Purpose: Satisfies ROADMAP Phase 7 success criterion #4 ("the state machine is input-shape agnostic"). Test files live next to the routes they test, sharing the existing `vi.hoisted + Proxy` mock pattern. Tests run independently of Plan 02 — only depend on Plan 01's schema migration.

Output: Two extended test files with new D-17 describe blocks; full Vitest suite still passes; D-17 invariant pinned.
</objective>
