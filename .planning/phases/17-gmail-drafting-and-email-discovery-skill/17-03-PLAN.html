Invocation (D-05): draft-outreach-emails <campaign-id> — batch-only, discover-then-draft in ONE run. NO --discover-only / --draft-only flags.

allowed-tools (D-06 allowlist — Gmail): mcp__gmail__search_threads, mcp__gmail__get_thread, mcp__gmail__create_draft, mcp__gmail__list_drafts. Plus Read, Bash. NO send/trash/update/delete tool exists or is allowed.

Discovery accept rule (D-03, LOCKED): query search_threads with the contact's full name → for each thread get_thread → walk messages[].payload.headers (From/To/Cc) → parse "Display Name " → strip steve@bronstein.org → keep only addresses whose display name case-insensitively matches the contact's first+last (Pitfall 3) → dedupe by lowercased address (Pitfall 4). Then: exactly 1 distinct → PATCH .../recipient {channel:'email', recipientEmail}; email joins the drafting queue this run. ≥2 distinct → AMBIGUOUS: leave recipientEmail UNSET (do NOT force linkedin_message), list contact+candidates in summary, skip drafting. 0 → PATCH .../recipient {channel:'linkedin_message'} (route nulls recipientEmail); list in linkedin-fallback.

Idempotency (D-02 / DRFT-03): the approved queue NEVER contains status='drafted' (already skipped before the loop). Per drafting item: gmailDraftId IS NULL → new draft. gmailDraftId IS NOT NULL (edited-after-draft → returned to approved with stale id) → create a FRESH draft + repoint via /draft (no update_draft tool exists; old draft left harmless, never sent).

Draft content (Pitfall 2): subject = editedSubject ?? generatedSubject; body = editedBody ?? generatedBody. If both null → skip + report (same as archived edge case).

Per-email failure (discretion): approved→failed is ILLEGAL. On a draft failure of an approved email do NOT call /status; leave it approved, add to failed list, continue. (Only call /status {failed,lastError} for a non-approved email, edge case.)

Archived contact (discretion): contact.archivedAt != null (already in the work-queue item) → skip drafting + report.
