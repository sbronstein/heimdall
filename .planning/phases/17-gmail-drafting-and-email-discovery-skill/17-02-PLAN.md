1. GET /api/outreach-campaigns/[id]/emails?status=approved&limit=100
   → { success, data: [{ email: OutreachEmail, contact: Contact|null }], meta: { cursor, hasMore } }
   email fields: id, campaignId, contactId, channel, recipientEmail, generatedSubject, generatedBody,
                 editedSubject, editedBody, status, gmailDraftId, lastError, lastErrorAt, draftedAt, ...
   contact fields: id, firstName, lastName, email, outreachStatus, archivedAt, linkedinUrl, title, currentCompany
   Default limit 20, MAX 100. Paginate via &cursor=<meta.cursor> until meta.hasMore=false.

2. PATCH /api/outreach-campaigns/[id]/emails/[emailId]/recipient
   body { channel: 'email', recipientEmail: '<addr>' }  → accept a discovered address (DISC-01/02)
   body { channel: 'linkedin_message' }                  → route forces recipientEmail=null (DISC-03 / D-04b)

3. PATCH /api/outreach-campaigns/[id]/emails/[emailId]/draft
   body { gmailDraftId: '<id>' } → (post-Plan-17-01) sets status='drafted' (canEmailTransition-gated),
   draftedAt, contact.outreachStatus='reached_out', logs timeline. Legal pre-state: approved ONLY.

4. PATCH /api/outreach-campaigns/[id]/emails/[emailId]/status
   body { status: 'failed', lastError: '<...>' } → FAILURE PATH ONLY, and ONLY for non-approved emails.
   approved→failed is ILLEGAL (returns 400). Do not call this for an approved email that fails to draft.
