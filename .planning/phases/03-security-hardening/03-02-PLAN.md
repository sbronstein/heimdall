<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server-side render → external HTTP (api.github.com) | The auth pages currently issue an SSR fetch to a third-party domain on every render. The third party can observe Heimdall's render-time pattern (low-value but unnecessary signal leak); the request itself runs in Next.js's SSR runtime. |
| User → "Continue with GitHub" button | The user is presented with what looks like a functional OAuth control. Clicking it produces a `console.log` only — no auth flow, no error toast, no visible failure. UX risk shaped like a phishing surface (the user believes a third auth method exists). |
