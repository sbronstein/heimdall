    Export one named async function: `callRoute(handler, opts)`. Its contract — these are the exact parameter shapes downstream tests will use:

    Type signature (TypeScript):
    ```
    type RouteHandler = (
      request: Request,
      ctx?: { params: Promise<Record<string, string>> }
    ) => Promise<Response>;

    type CallRouteOptions = {
      method?: string;        // default 'GET'
      url?: string;           // default 'http://localhost/test'
      body?: unknown;         // JSON-serialized into request body
      formData?: FormData;    // if provided, takes precedence over body and sets multipart
      params?: Record<string, string>;          // wrapped in Promise.resolve before passing to handler
      searchParams?: Record<string, string>;    // appended to url
      headers?: Record<string, string>;
    };

    export async function callRoute<T = unknown>(
      handler: RouteHandler,
      opts: CallRouteOptions = {}
    ): Promise<{ status: number; body: T; response: Response }>
    ```

    Implementation requirements:
    1. Build the final URL: start from `opts.url ?? 'http://localhost/test'`, append `opts.searchParams` as a query string using `URLSearchParams` if present.
    2. Build the `RequestInit`: method = `opts.method ?? 'GET'`. If `opts.formData` is present, set `body: opts.formData` and do NOT set `Content-Type` (the runtime fills the multipart boundary). Otherwise if `opts.body` is present, set `body: JSON.stringify(opts.body)` and `headers: { 'Content-Type': 'application/json', ...(opts.headers ?? {}) }`.
    3. Construct `const request = new Request(url, init)`.
    4. Invoke the handler. If `opts.params` is present, pass `{ params: Promise.resolve(opts.params) }` as the second argument (mirrors Next.js 16 async-params shape). Otherwise call `handler(request)` with no second argument.
    5. Await `response.json()` to get the body. If the body is not valid JSON, fall back to `await response.text()` and return the text typed as `T` (some routes may legitimately return non-JSON — keep it tolerant).
    6. Return `{ status: response.status, body, response }`.

    Do NOT add framework-specific imports beyond standard web APIs (`Request`, `Response`, `FormData`, `URLSearchParams`). Do NOT depend on `next/server` — node's global `Request` is what App Router route handlers actually receive.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dev-machine → no external | Test harness must not reach the network. PGlite is in-process; no Neon, no Clerk runtime, no external HTTP. |
| committed-test-code → repository | Test fixtures and config must not leak credentials. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information disclosure | `src/test-utils/pglite.ts` | mitigate | Use `new PGlite()` with no filesystem path → pure in-memory; do NOT read `process.env.DATABASE_URL`. PGlite has no network adapter, so even an accidental real-DB import is structurally impossible. |
| T-02-02 | Information disclosure | `vitest.config.ts` | mitigate | Do NOT load `.env.local` or `dotenv` in the vitest config. Tests run without any real secret being read. |
| T-02-03 | Spoofing | `src/test-utils/call-route.ts` | accept | Helper constructs synthetic `Request` objects only — no auth headers, no Clerk session. Auth-gate testing is owned by Phase 3 (SEC-A1) per 02-CONTEXT.md Deferred. Low risk in this phase. |
</threat_model>
