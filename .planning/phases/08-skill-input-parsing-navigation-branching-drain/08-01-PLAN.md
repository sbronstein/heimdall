<objective>
Extend `GET /api/job-leads` to expose `companyLinkedinUrl` as a top-level field on every lead row, sourced via a leftJoin to the `companies` table. This is the data Phase 8's skill drain loop needs to navigate directly to a company-scope lead's employees page without a per-lead `GET /api/companies/[id]` round-trip.

Purpose: Implements user decision D-13 ("Extend GET /api/job-leads to include companyLinkedinUrl as a single top-level field per lead row"). Without this field present in the response, the drain loop (Plan 08-03) cannot tell whether a company-scope lead has a persisted LinkedIn company URL or needs the mid-drain fallback (D-14).

Output:
- Modified `src/app/api/job-leads/route.ts` GET handler (POST handler untouched — Phase 7 owns it)
- Extended `src/app/api/job-leads/route.test.ts` with a single new test case pinning the new field on both lead types (CD-04)
- `npm test` passes; `npm run build` succeeds
</objective>
